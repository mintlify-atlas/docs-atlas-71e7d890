## What is CURD?

CURD is a code-intelligence control plane that helps you understand codebases structurally, mutate code safely, and author repeatable workflows. Built for serious solo developers, supervised agents, and larger agentic systems.

## Quick Start

Get CURD running in minutes with your preferred installation method.

<CodeGroup>

```bash Homebrew
brew tap aerospeedsta/curd && brew install curd
```

```bash Winget
winget install curd
```

```bash Scoop
scoop bucket add curd https://github.com/aerospeedsta/curd-scoop.git
scoop install curd
```

```bash AUR
paru -S curd-bin
```

```bash Python (uvx)
uvx --from curd-python curd --version
```

```bash Node.js (bunx)
bunx --bun curd-node --version
```

</CodeGroup>

## First Commands

Initialize CURD and explore a codebase in three commands:

```bash
curd init
curd search alpha
curd graph src/lib.rs::alpha --direction both --depth 2
```

## Core Capabilities

<AccordionGroup>
  <Accordion title="Code Intelligence" icon="brain">
    - **Ranked Search**: BM25/FTS-backed symbol indexing with Tree-sitter parsing
    - **Structural Reads**: URI-based reads with contextual mapping
    - **Graph Traversal**: Dependency and call graphs with topological sorts
    - **Multi-Language**: Rust, Python, JavaScript, TypeScript, Go, C/C++, Java
  </Accordion>

  <Accordion title="Safe Mutation" icon="shield">
    - **Shadow Workspace**: Physical shadow store for conflict-aware commits
    - **Transaction Flow**: Begin, diff, commit, or rollback changes safely
    - **Graph Audits**: Architectural alerts and semantic integrity post-commit
    - **Session Enforcement**: Mandatory sessions for all state-mutating operations
  </Accordion>

  <Accordion title="Workflow Automation" icon="gears">
    - **.curd Scripts**: Human-readable workflow language with structured comments
    - **Plan Artifacts**: Compiled execution contracts with bound arguments
    - **Profile Gating**: Capability atoms and runtime ceilings for controlled execution
    - **Explainability**: Preserve intent metadata from source to execution
  </Accordion>

  <Accordion title="Agent Integration" icon="robot">
    - **MCP Server**: Model Context Protocol server for AI agent integration
    - **Profiles**: Role-based capability gating (assist, supervised, autonomous)
    - **Runtime Ceilings**: Full and lite modes for different deployment contexts
    - **Policy Enforcement**: Blocklists, allowlists, and approval workflows
  </Accordion>
</AccordionGroup>

Most coding workflows force you to choose between fast-but-sloppy local automation or heavy-but-awkward governed automation. CURD keeps both layers:
